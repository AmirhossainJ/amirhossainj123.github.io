Introduce

So you want to make your own game, with coding, but you don't know much about coding?

you search on internet to find a good game making library for js, that is simple enough

or maybe ask a friend which introduces this game making library


And here we are introducing Electrame a simple Game Making2D library

which can make games so simply, you just got to know javascript a lil bit and

you can start making your own game already! this library obviously, isn't as advanced as threejs

cause its not supposed to be advanced, like the meaning of "simple game making library"

is that the user can make its own game simply, and making it simple kinda have limitations


How does it work?

So reading this part means you like our library, right? :D

well our library uses a thing in html called canvas, which is sadly not supported in firefox

canvas gives the programmers tools to make cool screens and stuff, but it might get annoying very soon

for game making. here you understand what you write, which is technically possible if you make your game with functions

but we did it for you so not needed, for example here you can just define a shape and pick a color

or just put a rgb color yourself there, but we put a bunch of premade and useful colors there for you so yep

and just merge those 2 things into a object, which will generate this informations with it too

Position X, Position Y, Velocity X, Velocity Y, (Scale X, Scale Y) or Radius, Id, Shape, Cid

Cid is the collision id, which for example an image has a collision id of a rectangle("Square" in the code)

which you never ever need to touch unless you want your image to collide differently which happens rarely and others are obvious

so how do we render stuff?

we draw the frame, then after the next frame we delete the old frame and create a new one

Note: our X and Y positioning and velocity system is, Increase X = to the right of the screen, Increase Y = to the bottom of the screen

Getting Started PART 1

So finally a new user wants to learn our library, AMAZING!

So first of all! we need to tell the library to be prepared to work, so just put this command there, EInit()

EInit();

then we add the canvas to html like the normal way OR we use ECreateGame(id,width,height) which we gonna name the game: first

and I think our game would be 600 by 300 so here it is

ECreateGame("first", 600, 300);

Then we can config if our world has gravity or friction

let Gravity = false; let Friction = true; EConfig(Gravity,Friction);

Then we can define our game! with EGame(id) which in this case is

let Game = new EGame("first");

Now we have our game! but we need a background right? first lets add the color of it, I am gonna go with bright blueDay color

so we can use EColors for premade stuff, so this is how it is EColors(opacity 1 to 0).color

let Background = new EColors(1).DAY;

You can check the materials tab for a full list of the colors you can also use RGB

So the color is set, but for the shape, we all know our background is rectangle but sometimes you might use css to make your game screen circle

but we not gonna do that now so lets define the shape of the background! we can use ESquare(width,height)

let Background_Resolution = new ESquare(600,300);

now lets define the object! (x,y,shape,material)

let Sky = new EObject(0,0,Background_Resolution,Background);

now we defined the background, you can try and define the ground yourself now too

so after we defined everything we define a function and render everything! (Game, Object)

function my_first_game() { ERender(Game,Sky); } my_first_game();

current code:

EInit(); ECreateGame("first", 600, 300); let Gravity = false; let Friction = true; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); function my_first_game() { ERender(Game,Sky); } my_first_game();

so enough for part one!


Getting Started PART 2

Oooo hi again :D

so in the last part we learned how to create an object and render it out!

now we can code something just like this!

EInit(); ECreateGame("first", 600, 300); let Gravity = false; let Friction = true; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); } my_first_game();

So we have drawn a simple sky and a ground and the player on top of the ground

I am gonna add some keys for the player to move, but before that we need to do a bit of physics stuff

I am gonna also turn on the gravity since we have a ground cause thats how usually games work?

But first off, our current thing is just a dead image, if we want to bring animation we have to loop it

to loop it, we not gonna use for or while or something like that, we need a slow loop so like this

we can setup a loop that runs every 10ms which is our frame rate speed! we can change fps that but we keep it like that for now

function example() { setTimeout(() => { // your code goes here! example(); }, 10); }

we also need to clear the screen every time we want to render again

another command introduced! EClear()

So if we implement the same thing we used in this and the other example and enable gravity we get this:

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { setTimeout(() => { EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); my_first_game(); }, 10); } my_first_game();

But as you see nothing is happening

So I introduce you another command! EApplyPhysicsTo(object)

So this command applies the physics you have just set in the config part! remmeber?

So maybe you wanted to turn off gravity and friction, you can just don't run this command!

So I also have to introduce another command which applies the motion of the object

Like if you apply a lot of force and gravity and friction, Bla bla bla, you need to ALSO run this command

or else non of them move, EApplyMotion(Object)

So lets just put these all together

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { setTimeout(() => { Player = EApplyMotion(Player); Player = EApplyPhysicsTo(Player); EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); my_first_game(); }, 10); } my_first_game();

So as you see there is a cube which falls and phase through the ground but we don't like that!

So we make the ground have collision! with introducing another command:

EApplyCollision(Dynamic Object, Static Object, Bounce Force[force right, force left, force bottom, force top])

I know it was hard but trust me its simple ;-;

Note: if you want 2 Dynamic object just run this command 2 times but change the place of the first arg with the second

Ok lets use this command!

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { setTimeout(() => { Player = EApplyMotion(Player); Player = EApplyCollision(Player,Ground,[0,0,0,0]); Player = EApplyPhysicsTo(Player); EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); my_first_game(); }, 10); } my_first_game();

As you see the player FINALLY stands on the ground and touch grass

But its boring to watch it stand on the ground, isn't it?

So I introduce you another command: EKeyboard(Key)

which checks if a key is pressed or not, yep pretty much

Also I introduce another command: EApplyForce(Object,Force[Force X, Force Y], Limit[Left, Top, Right, Bottom])

which applies a force to the object, but look its not confusing, the limit is just a thing

that won't let the force be applied if its at a certain speed or higher, you can just put false, there for no limit

So lets just add this to our code!

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { setTimeout(() => { Player = EApplyMotion(Player); Player = EApplyCollision(Player,Ground,[0,0,0,0]); Player = EApplyPhysicsTo(Player); if (EKeyboard("W")) Player = EApplyForce(Player,[0,-1],[3,3,-3,-3]); if (EKeyboard("A")) Player = EApplyForce(Player,[-0.2,0],[3,3,-3,-3]); if (EKeyboard("S")) Player = EApplyForce(Player,[0,0.2],[3,3,-3,-3]); if (EKeyboard("D")) Player = EApplyForce(Player,[0.2,0],[3,3,-3,-3]); EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); my_first_game(); }, 10); } my_first_game();

So as you see now you can move perfectly but the problem is when you try to jump it like flies idk

Getting Started PART 3

So you want to fix the problem if it jumping every time huh?

first you can setup a variable called jump_able which is false on default but when the object

collides with something it enables it! and if its not colliding, disables it

to check if 2 rectangles are colliding you can just simply use EBoxCollision(Object 1, Object 2)[0]

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; let jump_able = false; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { setTimeout(() => { Player = EApplyMotion(Player); if (EBoxCollision(Player,Ground)[0]) jump_able = true; else jump_able = false; Player = EApplyCollision(Player,Ground,[0,0,0,0]); Player = EApplyPhysicsTo(Player); if (EKeyboard("W") && jump_able) Player = EApplyForce(Player,[0,-3],[3,3,-3,-3]); if (EKeyboard("A")) Player = EApplyForce(Player,[-0.2,0],[3,3,-3,-3]); if (EKeyboard("S")) Player = EApplyForce(Player,[0,0.2],[3,3,-3,-3]); if (EKeyboard("D")) Player = EApplyForce(Player,[0.2,0],[3,3,-3,-3]); EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); my_first_game(); }, 10); } my_first_game();

And yep now its time to not let the player escape the scene cause we don't see it anymore then ;c

So ye now you got to learn bunch of lil commands


EVelocityX(object) = returns the velocity in x direction of object

EVelocityY(object) = returns the velocity in y direction of object

EPositionX(object) = returns the position x of object

EPositionX(object) = returns the position y of object

EScaleX(object) = returns the scale x of object(image or rectangle)

EScaleY(object) = returns the scale y of object(image or rectangle)

ERadius(object) = returns the radius of object(circle)

EType(object) = returns the type of object (Rectangle = "Square", Image = "Image", Circle = "Circle")


so these were the information ones now for the set ones


ESetVelocityX(object,value) = sets the velocity in x direction of object

ESetVelocityY(object,value) = sets the velocity in y direction of object

ESetPositionX(object,value) = sets the position x of object

ESetPositionX(object,value) = sets the position y of object

ESetScaleX(object,value) = sets the scale x of object(image or rectangle)

ESetScaleY(object,value) = sets the scale y of object(image or rectangle)

ESetRadius(object,value) = sets the radius of object(circle)


But why do we exactly need these? well in order to not let the player escape the screen you have to teleport it back

and teleportation means change of position not speed or velocity, and the rest might get handy for you later

so first we have to check if the player is not colliding with the Sky then teleport so here we go:

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; let jump_able = false; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); function my_first_game() { setTimeout(() => { Player = EApplyMotion(Player); if (EBoxCollision(Player,Ground)[0]) jump_able = true; else jump_able = false; if (!(EBoxCollision(Player,Sky)[0])) { ESetPositionX(Player,290) ESetPositionY(Player,240) } Player = EApplyCollision(Player,Ground,[0,0,0,0]); Player = EApplyPhysicsTo(Player); if (EKeyboard("W") && jump_able) Player = EApplyForce(Player,[0,-3],[3,3,-3,-3]); if (EKeyboard("A")) Player = EApplyForce(Player,[-0.2,0],[3,3,-3,-3]); if (EKeyboard("S")) Player = EApplyForce(Player,[0,0.2],[3,3,-3,-3]); if (EKeyboard("D")) Player = EApplyForce(Player,[0.2,0],[3,3,-3,-3]); EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); my_first_game(); }, 10); } my_first_game();

So it teleports the player to the center of the screen in x and in y, just on ground

So now if we add bunch more objects we can create a very simple jump type of game, which is what we gonna do!

So now I am gonna just add a box that player can jump on

now we gotta add another apply collision and jump_able rule!

EInit(); ECreateGame("first", 600, 300); let Gravity = true; let Friction = true; let jump_able = false; EConfig(Gravity,Friction); let Game = new EGame("first"); let Background = new EColors(1).DAY; let Background_Resolution = new ESquare(600,300); let Sky = new EObject(0,0,Background_Resolution,Background); let Grass = new EColors(1).GREEN; let Ground_Resolution = new ESquare(600,50); let Ground = new EObject(0,250,Ground_Resolution,Grass); let Player_Color = new EColors(1).BLACK; let Player_Shape = new ESquare(10,10); let Player = new EObject(20,230,Player_Shape,Player_Color); let Pad_Color = new EColors(1).BLUE; let Pad_Shape = new ESquare(50,10); let Pad = new EObject(20,220,Pad_Shape,Pad_Color); function my_first_game() { setTimeout(() => { Player = EApplyMotion(Player); if (EBoxCollision(Player,Ground)[0] || EBoxCollision(Player,Pad)[0]) jump_able = true; else jump_able = false; if (!(EBoxCollision(Player,Sky)[0])) { ESetPositionX(Player,290) ESetPositionY(Player,240) } Player = EApplyCollision(Player,Ground,[0,0,0,0]); Player = EApplyCollision(Player,Pad,[0,0,0,0]); Player = EApplyPhysicsTo(Player); if (EKeyboard("W") && jump_able) Player = EApplyForce(Player,[0,-3],[3,3,-3,-3]); if (EKeyboard("A")) Player = EApplyForce(Player,[-0.2,0],[3,3,-3,-3]); if (EKeyboard("S")) Player = EApplyForce(Player,[0,0.2],[3,3,-3,-3]); if (EKeyboard("D")) Player = EApplyForce(Player,[0.2,0],[3,3,-3,-3]); EClear(Game); ERender(Game,Sky); ERender(Game,Ground); ERender(Game,Player); ERender(Game,Pad); my_first_game(); }, 10); } my_first_game();
